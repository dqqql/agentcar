# 智能座舱出行规划 Agent

## 项目简介

本项目面向“智能座舱出行规划”场景，目标是做一个可演示、可扩展的 MVP：

`用户输入需求 -> 提取目的地与约束 -> 拉取景点/餐厅/酒店候选池 -> 客观加权排序 -> 输出前 N 个组合 -> 前端展示`

当前项目仍处于早期执行阶段，已经完成最小后端启动，并补齐了景点、餐厅、酒店三类数据抓取脚本的基础能力。

## 当前阶段

当前推荐用“四层流程”来理解项目进度：

1. 输入与提取
2. 客观数据提取
3. 客观加权算法
4. 前端输出

目前真实进度是：

- 第一层：方案已明确，代码未启动
- 第二层：进行中，三类数据脚本已具备基础能力
- 第三层：未开始
- 第四层：未开始

详细进度见：

- [doc/流程图版开发进度.md](doc/流程图版开发进度.md)

## 当前已完成内容

- 项目基础目录初始化完成
- FastAPI 最小服务已启动
- 后端基础接口：
  - `/`
  - `/health`
- `scripts/getdata/place/main.py` 可抓取景点/地点类数据
- `scripts/getdata/food/main.py` 可抓取餐饮类数据
- `scripts/getdata/hotel/main.py` 可生成酒店候选原型数据
- 三类脚本输出结构已统一为：
  - `output/<dataset>_<query>_<timestamp>/summary.csv`
  - `output/<dataset>_<query>_<timestamp>/detail.json`

## 当前不做什么

现阶段不优先做这些内容：

- 真实座舱接入
- 语音输入与语音转文本
- 深度学习推荐
- 多轮复杂对话
- 实时价格查询
- 复杂路线优化

MVP 先收敛到“文本输入 + 客观数据 + 规则排序 + 基础展示”。

## 目录说明

- `backend/`：后端代码
- `frontend/`：前端演示页
- `data/`：业务数据与 mock 数据
- `scripts/getdata/`：客观数据抓取与原型数据生成脚本
- `doc/`：项目文档与进度记录
- `tests/`：测试代码

## 数据脚本说明

`scripts/getdata` 下目前有三类脚本：

- `place/`：地点 / 景点类数据抓取
- `food/`：餐饮类数据抓取
- `hotel/`：酒店候选数据生成

三个脚本现在都使用统一的输出目录结构，各自产物都写到本目录下的 `output/` 中。仓库默认忽略 `output/`、`__pycache__/` 和历史导出数据，保证其他人拉取代码时只拿到脚本本身。

## 本地运行

在项目根目录执行：

```bash
uvicorn backend.app.main:app --reload
```

启动后访问：

- [http://127.0.0.1:8000/](http://127.0.0.1:8000/)
- [http://127.0.0.1:8000/health](http://127.0.0.1:8000/health)
