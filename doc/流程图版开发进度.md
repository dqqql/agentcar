## 1. 流程图分层定义

### 第一层：输入与提取

- 用户输入方式：文本输入为当前 MVP 主入口，语音输入为后续扩展入口
- 输入处理能力：包括语音转文本、命名实体识别、目的地提取、约束信息提取
- 当前目标：从用户一句自然语言里提取出后续数据抓取和排序所需字段

### 第二层：客观数据提取

- 景点池：等级、流行度、门票价格等客观字段
- 餐厅池：类型、星级、人均价格等客观字段
- 酒店池：排名、评分、评论数、房价区间等客观字段
- 当前目标：先保证三类池子都能稳定产出结构化样本

### 第三层：客观加权算法

- 指标归一化：将不同量纲的价格、评分、距离、热度统一到可比较区间
- 加权得分计算：按权重计算综合得分
- 空间聚类 / 关联过滤：处理距离冲突、密度聚集和组合可行性

### 第四层：前端输出

- 输出形态 1：前 N 个组合列表
- 输出形态 2：前端网页渲染展示
- 当前目标：先有清晰可读的列表和结果页，再考虑视觉增强

## 4. 按流程图拆分的阶段目标

### 阶段 A：先跑通第一层到第二层

- 输入一句文本需求
- 提取目的地、景点偏好、餐饮偏好、预算约束
- 拉取或读取对应的景点池、餐厅池、酒店池

阶段完成标准：
- 后端能接收一段自然语言文本
- 后端返回结构化提取结果
- 后端能基于提取结果拿到三类候选数据

### 阶段 B：补第三层客观加权算法

- 对价格、评分、热度、距离等指标归一化
- 形成一套可解释的权重表
- 输出综合分和排序结果

阶段完成标准：
- 每条景点、餐厅、酒店都有明确得分
- 能根据规则选出前 N 个候选组合

### 阶段 C：补第四层输出闭环

- 后端输出 Top N 组合列表
- 前端展示列表、组合摘要、基础详情

阶段完成标准：
- 用户能在页面输入需求
- 页面能展示解析结果、候选组合和说明

### 阶段 D：再补语音与空间优化

- 接入语音输入
- 增加语音转文本
- 增加空间聚类 / 关联过滤优化

阶段完成标准：
- 支持语音或文本两种入口
- 结果组合明显优于纯规则硬拼接
