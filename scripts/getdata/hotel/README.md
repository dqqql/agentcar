# 酒店原型数据集

该脚本用于构建一个酒店候选数据集，供算法原型设计与验证使用。

功能说明：

1. 使用 OpenStreetMap 的 Nominatim 对目的地进行地理编码。
2. 通过 OpenStreetMap Overpass 拉取附近公开的酒店 POI 数据。
3. 保留酒店名称、坐标、地址、网站和距离等公开字段。
4. 根据请求的入住日期，生成可复现的伪造房量、房型、价格和可订状态数据。

输出文件：

- `*_summary.csv`：扁平化酒店列表，便于快速筛选或排序
- `*_detail.json`：包含酒店、房型和每日报价的嵌套数据，适合算法设计使用

运行方式：

```powershell
python scripts/getdata/hotel/main.py
```

建议用途：

- 将汇总文件输入到第一阶段的酒店排序模型中
- 使用详细 JSON 做房型匹配、库存压力评分和基于价格的推荐规则设计
